eject_client

  • eject_client(client, disconnect_code, reason): void

  • Ejects the client from the Astron virtual world with a reason.

    Parameters

    • client: bigint

      The channel ID (uint64) of the client to eject.

    • disconnect_code: number

      The disconnect code to send to the Astron Client Agent.

    • reason: string

      A string message describing the reason for the client eject.

    Returns void

    Remarks

    You may be confused why this is accessible to a ClientRepository on the NodeJS implementation. The reason I did this was to keep a single DistributedObject class for all repositories to use. This method will perform a check to make sure it's repository is of InternalRepository before continuing. If not, it throws the SuspiciousClientBehavior error. (Either way, the Astron CA will eject the client if it tries to send this message) This method will be removed in the chrome implementation, since no Internal repositories will run on the browser.
